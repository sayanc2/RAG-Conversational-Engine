🤖 The 5-Agent Ensemble

🎭 ORACLE Conductor

Model: Claude Sonnet 4.5

Role: Main orchestrator that receives all queries, detects intent, routes to specialists or calls tools, and composes final answers.

Handoff: HERALD, ARCHIVIST Tools: herald_as_tool, archivist_as_tool

🌤️ HERALD

Model: Claude Sonnet 4.5

Role: Fetches live weather and news via Tavily API, embeds into ChromaDB.

tavily_weather_fetch tavily_news_search embed_and_store_live_context

👤 ARCHIVIST

Model: Claude Sonnet 4.5

Role: Queries SQLite employee database and performs semantic location mapping via Chroma.

sql_query_employee semantic_location_mapper

✅ VALIDATOR

Model: GPT-4o (independent perspective)

Role: Post-generation groundedness critic. Verifies every claim against sources.

Score calculation: grounded_claims / total_claims Threshold: <0.70 = HITL

🛡️ SENTINEL

Model: GPT-4o-mini (fast & cheap)

Role: Dual-mode security. INPUT: Detects injection/PII. OUTPUT: Detects leaks.

Runs in parallel Pre-generation + post-generation

⚡ Canonical Request Flow

Query: "What is the weather like where Raghav works?"

  1. SENTINEL Input Guardrail (parallel, fast) → Checks for prompt injection/PII extraction → PASS
  2. Conductor Intent Detection → Recognizes blended query (employee + weather)
  3. Archivist Tool Call → Runs `sql_query_employee(name="Raghav")` → Returns EMP-0042, Austin TX, Engineering
  4. Herald Tool Call → Runs `tavily_weather_fetch("Austin, TX")` → Gets 94°F, Partly Cloudy → Embeds into live_context Chroma collection
  5. Semantic Location Bridge → `semantic_location_mapper("Austin, TX")` checks employee_locations collection → Confirms Raghav @ Austin
  6. Conductor Composition → Blends both results: "Raghav Sharma works in Austin, TX. Current weather: 94°F, Partly Cloudy."
  7. SENTINEL Output Guardrail → Checks for PII leak (name+location combo) → PASS
  8. VALIDATOR Critic → Verifies claims against sources → Score 0.97 (pass) → Proceeds
  9. Final Response → Answer + sources ([SQL: EMP-0042], [Tavily: Austin weather]) delivered to UI with 97% confidence badge

🔀 Handoff vs Tool Pattern

When does each agent act as FULL AGENT (Handoff) vs TOOL (Tool Call)?

Agent As Full Agent (Handoff) As Tool (Tool Call)
HERALD Pure weather/news query: "What's the weather in Austin?" Blended query: Called by Conductor alongside archivist_as_tool
ARCHIVIST Pure employee query: "Find all engineers in Austin" Blended query: Called by Conductor alongside herald_as_tool
VALIDATOR Not typically used as full agent As output_guardrail + optional on-demand re-validation tool
SENTINEL Not used as full agent As input_guardrail + output_guardrail (always attached)

🛡️ Dual Guardrail Safety Model

Three-Layer Defense

Layer 1: Input Guardrail (SENTINEL)

Runs in parallel with Conductor startup (low latency).

Violations detected: prompt_injection, off_topic, pii_request, jailbreak, data_exfiltration

Action: If severity=high → InputGuardrailTripwireTriggered → User sees friendly rejection

Layer 2: Output Groundedness Check (VALIDATOR)

Post-generation critic using GPT-4o (different model = no confirmation bias).

Score calculation: grounded_claims / total_claims

Score Range Verdict Action
≥ 0.85 PASS Publish directly
0.70 - 0.85 WARN Publish + log warning
< 0.70 FAIL Activate HITL panel

Layer 3: Output Guardrail (SENTINEL)

Runs on final response before user sees it.

Violations: pii_leak (multiple employees' names+age+location), data_exfiltration (bulk dumps)

Action: Block or log based on severity

👥 Human-In-The-Loop (HITL)

When HITL Triggers:

  • Groundedness Score < 0.70 (VALIDATOR fail verdict)
  • Sensitive PII Combination (detected by Conductor or Sentinel)

HITL Panel Components:

📊 Visual Indicators

• Groundedness bar (🟢 pass / 🟡 warn / 🔴 fail)

• Percentage score (0-100%)

• Verdict label

🔍 Ungrounded Claims

List of specific claims that could not be verified against sources.

Example: "Temperature is 94°F" (not in Tavily results)

✏️ Editable Draft

Text area pre-filled with AI draft. Human can edit before approving.

🎯 Action Buttons

✅ Approve (publish as-is)

✏️ Edit & Approve (publish edited)

🔄 Regenerate (re-run with error feedback)

Logging & Analytics:

• Human-corrected answers logged separately

• Regenerate count tracked

• Session saves to SQLite after HITL resolution
